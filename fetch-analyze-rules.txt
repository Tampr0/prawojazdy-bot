SPEC: ANALIZA FETCH LOG
1. DEFINICJE
1.1 Success event
Success event = linia zawierająca:
FETCH_OK
FETCH_RECOVERED
Oba traktowane identycznie.

1.2 Ignorowane eventy
Ignorujemy całkowicie:
FETCH_RETRY_X
FETCH_FAILED (nie tworzy success event)

2. STRUMIEŃ DANYCH
Z loga budujemy sekwencję:
S = [t1, t2, t3, ... tn]
gdzie:
ti = timestamp każdego success eventu (OK lub RECOVERED)

3. METRYKA GŁÓWNA
3.1 Odstęp między fetchami
Dla każdego i:
delta_i = t_i - t_(i-1)

3.2 Średnia
avg_delta = (Σ delta_i) / (n-1)

3.3 Mediana
Sortujemy:
delta_sorted = sort(delta_i)
Mediana:
jeśli n parzyste → średnia 2 środkowych
jeśli nieparzyste → środkowy element

4. METRYKI WAF
Liczymy tylko dla eventów FETCH_RECOVERED

4.1 Retry count
Dla każdego FETCH_RECOVERED:
retry_count = liczba FETCH_RETRY_X bezpośrednio przed nim
Średnia:
avg_retry = (Σ retry_count) / liczba_recovered

4.2 Delay WAF
Z linii:
FETCH_RECOVERED ... totalDelay=XXXX
zbieramy:
delay_i = totalDelay

Średnia:
avg_delay = (Σ delay_i) / liczba_recovered
Mediana:
 analogicznie jak w pkt 3.3

5. ZASADY OBRÓBKI
5.1 Ignorujemy duration
Nie używamy:
duration=...

5.2 Retry nie tworzy punktu w czasie
Sekwencja:
OK → RETRY → RETRY → RECOVERED
oznacza:
jeden punkt końcowy = RECOVERED

5.3 FAILED
nie jest success event
nie generuje delta
traktowany jako brak danych

5.4 Jednostki
Wszystkie obliczenia:
timestamp → sekundy
delay → sekundy (jeśli w ms → dzielić przez 1000)

6. OUTPUT (TABELA)
Tabela zawiera wyłącznie:
INTERWAŁ_ODPYTYWANIA_MS
OPÓŹNIENIA_RETRY_MS
ŚREDNI_CZAS_MIĘDZY_FETCHAMI_S
MEDIANA_CZASU_MIĘDZY_FETCHAMI_S
ŚREDNIA_LICZBA_RETRY
ŚREDNIE_OPÓŹNIENIE_WAF_S
MEDIANA_OPÓŹNIENIA_WAF_S


7. CEL ANALIZY
Optymalizacja:
min(avg_delta)
przy stabilnym avg_retry i avg_delay
