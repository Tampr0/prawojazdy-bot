SPECYFIKACJA – ANALIZA FETCH LOGÓW
1. DEFINICJE ZDARZEŃ
Sukces (SUCCESS)

Zdarzenie uznawane za udane:

FETCH_OK
FETCH_RECOVERED
Retry (RETRY)

Zdarzenia pośrednie:

FETCH_RETRY_1
FETCH_RETRY_2
FETCH_RETRY_3
FETCH_RETRY_4
...
Fail (FAILED)

Zdarzenie nieudane:

FETCH_FAILED
2. PODSTAWOWA JEDNOSTKA ANALIZY
PRÓBA (ATTEMPT)

Jedna próba = pełny cykl:

start: pierwszy FETCH_RETRY_* lub bezpośrednio FETCH_OK
koniec:
FETCH_OK → sukces bez retry
FETCH_RECOVERED → sukces po retry
FETCH_FAILED → porażka
3. LICZBA WSZYSTKICH PRÓB
Definicja:

Każdy cykl zakończony jednym z:

FETCH_OK
FETCH_RECOVERED
FETCH_FAILED
Wzór:
TOTAL_ATTEMPTS = count(FETCH_OK + FETCH_RECOVERED + FETCH_FAILED)
4. LICZBA UDANYCH PRÓB
Definicja:
SUCCESS_COUNT = count(FETCH_OK + FETCH_RECOVERED)
5. PROCENT UDANYCH PRÓB
Wzór:
SUCCESS_RATE = (SUCCESS_COUNT / TOTAL_ATTEMPTS) * 100%
6. ROZKŁAD PO LICZBIE RETRY

Dotyczy tylko:

FETCH_RECOVERED retries = X
Metryki:
SUCCESS_AFTER_1_RETRY = count(FETCH_RECOVERED retries=1)
SUCCESS_AFTER_2_RETRY = count(FETCH_RECOVERED retries=2)
SUCCESS_AFTER_3_RETRY = count(FETCH_RECOVERED retries=3)
SUCCESS_AFTER_4_RETRY = count(FETCH_RECOVERED retries=4)
...
7. SUKCES BEZ RETRY
Definicja:
SUCCESS_NO_RETRY = count(FETCH_OK)
8. ŚREDNI CZAS MIĘDZY UDANYMI FETCHAMI (KLUCZOWA METRYKA)
Zasada:

Liczymy tylko między kolejnymi sukcesami:

SUCCESS = FETCH_OK lub FETCH_RECOVERED
Algorytm:
Wyciągnij wszystkie sukcesy w kolejności czasowej:
S1, S2, S3, ..., Sn
Dla każdego sąsiedniego:
INTERVAL_i = timestamp(S_i+1) - timestamp(S_i)
Wynik:
AVG_SUCCESS_INTERVAL = suma(INTERVAL_i) / (n - 1)
WAŻNE ZASADY:
1. Retry są ignorowane jako punkty
FETCH_RETRY_* nie tworzą interwałów
2. FETCH_RECOVERED = sukces
traktowane identycznie jak FETCH_OK
3. Start loga od retry

Jeśli log zaczyna się od:

FETCH_RETRY_*

→ ignorujemy aż do pierwszego:

FETCH_RECOVERED

→ to jest pierwszy punkt startowy

4. FETCH_FAILED wlicza się w czas

Jeśli między dwoma sukcesami występuje:

FETCH_FAILED

→ czas normalnie leci

→ NIE przerywa interwału

9. LICZBA NIEUDANYCH PRÓB
Definicja:
FAILED_COUNT = count(FETCH_FAILED)
10. CZAS TRWANIA NIEUDANYCH PRÓB

Z loga:

FETCH_FAILED totalTime=XXXXms
Metryki:
FAILED_TOTAL_TIME = suma(totalTime)
FAILED_AVG_TIME = średnia(totalTime)
11. OPCJONALNE (ALE PRAKTYCZNE)
Średnia liczba retry na sukces
AVG_RETRY_PER_SUCCESS =
    suma(retries z FETCH_RECOVERED) / SUCCESS_COUNT
12. FINALNY ZESTAW METRYK (DO OUTPUTU)

Minimalny zestaw:

TOTAL_ATTEMPTS
SUCCESS_COUNT
SUCCESS_RATE (%)

SUCCESS_NO_RETRY
SUCCESS_AFTER_1_RETRY
SUCCESS_AFTER_2_RETRY
SUCCESS_AFTER_3_RETRY
SUCCESS_AFTER_4_RETRY

FAILED_COUNT

AVG_SUCCESS_INTERVAL (s)

Opcjonalnie:

FAILED_AVG_TIME (ms)
AVG_RETRY_PER_SUCCESS
13. KLUCZOWA INTENCJA SYSTEMU

Ten system mierzy:

👉 realną przepustowość bota w warunkach WAF

a nie:

szybkość pojedynczego requestu
duration
latency API
14. NAJWAŻNIEJSZA METRYKA
AVG_SUCCESS_INTERVAL

Interpretacja:

niższy = bardziej agresywny i skuteczny bot
wyższy = więcej blokad / retry / WAF
